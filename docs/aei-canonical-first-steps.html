1. Overview

AEI (Artificial Embodied Intelligence) is a framework for working with movement archives that prioritizes activation over generation. Unlike generative AI systems that synthesize novel content, AEI focuses on transforming, analyzing, and discovering relationships within existing recorded movement.

This document describes the canonical first steps for implementing an AEI-compliant system.

2. Core Principles

2.1 Non-Generative Operation

AEI systems do not create new dances. They operate on recorded movement data, performing transformations such as:

  • Format conversion (video → skeletal coordinates → QTC sequences)
  • Feature extraction (identifying movement phrases, rhythmic patterns, spatial trajectories)
  • Similarity detection (finding correspondences between dances)
  • Visualization (rendering movement data in comprehensible forms)

2.2 Ethical Metadata

Every movement record in an AEI-compliant archive carries metadata specifying:

  • consent_level: What permissions were granted for this recording
  • attribution: Required credits for any use
  • permitted_uses: Array of allowed operations (research, education, visualization, etc.)
  • restricted_uses: Explicitly prohibited operations
  • community_contact: How to reach source communities for additional permissions

2.3 Interpretive Humility

AEI systems flag findings as hypotheses, not facts. Output always includes confidence levels and suggests avenues for human verification.

5. Ethical Checklist

Before any AEI operation, verify:

  1. Movement data has appropriate consent metadata
  2. Intended operation is within permitted uses
  3. Attribution requirements are met in any output
  4. Source communities have been consulted for sensitive analyses
  5. Findings are presented as hypotheses, not conclusions
